# 1. Web前端开发之上手伊始

&emsp;&emsp;互联网上有千千万万个网站，每个网站又由许多页面组成。开发大型商用网站是我们的终极目标，我们的起点就是要从做好地一个页面开始。

## 1.1 最简配置：你的小米加步枪

&emsp;&emsp;在开始做你的第一个页面之前，你需要确定你已经准备好了下面清单中所列出的工具：

### a. 一台连接到互联网的台式或者笔记本电脑

&emsp;&emsp;你准备的这台电脑的操作系统可以是微软windows系列中的任意一款（包括win7、win8和win10），也可以是在苹果公司的Macintosh系列电脑上的Mac OS系统（包括OS X、Mac OS 9、Mac OS 8）。当然如果你本人是开源操作系统的发烧友，你也可以选择任意一个类型的linux桌面版操作系统。

&emsp;&emsp;请特别留意：在决定选用任意一款操作系统之前，请你一定首先确认，这些系统不至于太老太旧，以至于它不能为你提供良好的上网体验。

###  b. 一个Web浏览器

&emsp;&emsp;Web浏览器是所有网站的客户端。

&emsp;&emsp;除了你现在看网页正在正用到的浏览器，你还可以为自己即将开始的Web前端学习选择一款专业的浏览器，通常这种给开发者用的浏览器都会提供专业而且强大的开发者工具（developer tools），比如说我们之前提到过的Google Chrome和Mozilla FireFox。

# 3. HTML，不得不说的事儿

&emsp;&emsp;刚才我们创建了一个显示最简单内容的小页面。

&emsp;&emsp;然而这样的网页实在只有用来演示的价值：用户不会从这样的页面上了解到任何新鲜资讯，企业也不会因这个页面而断定你可以承担网站开发的相关职责。如果是第一次看这篇文章，请你明确，这个教程的产生建立在这个假设之上，即：你之前只是听说过HTML这个词，但对于什么是HTML这样的问题，你没有深入思考过，更没有系统学习过。

&emsp;&emsp;制作这个小页面的过程，仅仅是让作为初学者的你，对于什么是网页有留下初步印象。

&emsp;&emsp;HTML的完整名称是**Hyper Text Markup Language**，中文全名是“**超文本标记语言**”，它结合了“超文本”和“标记语言”两种技术、两种特性。

 ![split_html_word](/public/img/html/split_html_word.jpg){:width="200px" height="300px"} 

## 3.1 超文本的来由

&emsp;&emsp;我们来做一个思想实验。

### 不需要翻书的时代

当我们中国人的老祖宗还在龟甲兽骨上刻字的时候，那个时候的人是怎么阅读的？

 ![jiaguwen](/public/img/html/jiaguwen.jpg){:width="200px" height="300px"} 

&emsp;&emsp;你一定会毫不犹豫，脱口而出：一篇文章才这么几个字，当然是一字不漏的按照书写顺序看下来啦！没错，那个时候不光是人类的发明的文字个数有限，一篇文章包含的字数也很有限，甚至这个世界上能够算得上文章并且可以流传开来的作品也非常稀缺。

&emsp;&emsp;用咱们的话说，古人的文章，那哪里是文章，都还没有现在的小学生写的练习作文长！换句话说，古代能够读书的人一般都是贵族或者富豪，他们也算是那个时代里面头脑最聪明的人，你能这么想，他们当然也会想得到。


### 手工翻书的日子

&emsp;&emsp;后来文字越来越多，有能力读书写字的人越来越多，它们写出来的文章越来越多，甚至结集成书，流传开来。最有趣的现象是，作者某甲出书的时候，书里面的内容说经常要用到另外一个作者书里面的词语解释、历史典故或者名人名言。这时，他就得给增加字词含义解释，说明典故的出处，甚至还要把被加上被引用文字部分或者全部内容了！

 ![tangshisanbaishou](/public/img/html/tangshisanbaishou.jpg){:width="400px" height="500px"}     

&emsp;&emsp;以纸质书为例。在人们还没有开始在电子设备上看报读书的年代，大家只能看到纸质书报。如果你足够细心，你肯定还记得文章作者们怎么给某一个名词编写自己的说明，或者怎么给某一个理论观点指明参考文献。每当他们发现需要这么做的时候，他一般会在自己的赐予
段落旁边加个小标记，这样读者就可以根据标记找到详细内容。

&emsp;&emsp;书页下方的名词解释：

 ![note_markup_paperbook](/public/img/html/note_markup_paperbook.png)

&emsp;&emsp;正文中的注释标记：

 ![note_markup_paperbook](/public/img/html/paper_book_ref_mark01.jpg){:width="300px" height="400px"}     

&emsp;&emsp;编辑在文章结尾补充的引用文献：

 ![note_markup_paperbook](/public/img/html/paper_book_ref_mark02.jpg){:width="300px" height="400px"} 

&emsp;&emsp;如果还需要证实被引用文献的真实性或者想要了解它们的更详细内容，你就得专门跑一趟路，去图书馆拿到被引用作品的完整纸质版本、手工查找。这种在书里面增加注释、声明引用，然后让读者手工翻书、跑腿查资料的情况直到现在还被继续沿用。

### 自由跳转的文本

&emsp;&emsp;自古至今，书籍都是捧起来读的。自从进入计算机时代之后，人们发现把传统的纸质文档、书籍用电子格式保存有很多好处。比如说，可以存更多内容，更省空间，更容易携带。然而，书多了，书和书之间的引用关系也越来越复杂。再去手工翻书的话，在时间上肯定不合算，而且人的宝贵精力十分有限。

&emsp;&emsp;电子书的大行其道，同时也从根本上扭转了人们“翻阅”图书的习惯。

 ![reading_woman](/public/img/html/reading_woman.jpg){:width="300px" height="400px"} 

&emsp;&emsp;出现在电子文档里面超文本，简单地说其实就是一段可以点击的文字：人只要轻轻动手一点，文档显示界面就会自动跳到另外一个位置上，这个目标位置通常就是作者希望你看到的、跟之前的可点击文字存在紧密关系的详细内容。

 ![thema_hypertext3](/public/img/html/thema_hypertext3.gif){:width="300px" height="400px"} 

&emsp;&emsp;人们把超文本的想法和技术引入网页，主要就是为了在网页之间建立快速链接。

### 网页上的超文本

&emsp;&emsp;我们来看一个具有历史意义的一个网站，它诞生于1991年8月6日，是[世界上第一个网站](http://info.cern.ch/hypertext/WWW/TheProject.html),准确地说，就在点开这个链接的时候，你已经体验过一次什么是超文本了：

 ![firstwebsite](/public/img/html/firstwebsite.png)

&emsp;&emsp;页面上所有显示为天蓝色并且带有下划线的文字，都是超文本。你用鼠标点击之后，浏览器窗口会跳转到了网站作者想要展现给你的页面上去。请你不要心存任何顾虑、大胆动手！我保证不会什么意外也不会发生！

## 3.2 什么是markup

 ![shougongpaiban](/public/img/html/jiaozhunshougao.jpg){:width="300px" height="400px"} 

&emsp;&emsp;“标识（markup）”这个词最早来源于传统出版业的“标记”一个手稿的说法：在原稿的边缘加注一些符号来指示打印上的要求。长久以来，这个工作都是由专门的人（"markup men"）以及校对人来进行，对原稿标志出使用什么样的字体，字型以及字号，然后再将原稿交给其它人进行手工的排版工作。

 ![shougongpaiban](/public/img/html/shougongpaiban.jpg){:width="300px" height="400px"} 


## 3.3  HTML应运而生

&emsp;&emsp;现今所有标记语言的“祖先”，标准通用标记语言(SGML，standard general Markup Language)诞生于1986年，是一种可以定义标记语言的元语言，同时也是一个ISO标准（标准编号是**ISO 8879:1986**）。它被用来给《牛津英语词典》电子书排版。然而，正因为它是世界上第一款标记语言，很多人在刚开始用的时候就发现它最大的问题：难学、难用。

 ![birth_of_html](/public/img/html/birth_of_html.jpg)

&emsp;&emsp;HTML之所以被创造出来，主要是为了给学不会SGML的人一个替代，他们可以依照这个规范更加便捷高效地编写、分发科学技术方面的电子资料。它的语法脱胎于SGML，但是更加简单易学、容易普及。随后很短时间内，就是这个HTML的发明者，一鼓作气用这种新型语言做出来了世界上第一个网站。

&emsp;&emsp;大家来认识一下他，蒂莫西·约翰·伯纳斯-李（Tim Berners-Lee）爵士，昵称蒂姆·伯纳斯-李，英国计算机科学家，麻省理工学院教授，同时也是万维网的发明者：

 ![creator_of_web](/public/img/html/creator_of_web.jpg)

&emsp;&emsp;1991年底，李爵士写了一个完整描述HTML语言的文档，并且公开到WWW网上。下面一段文字介绍了当时文章所描述的HTML语言的基本情况：

>It describes 18 elements comprising the initial, relatively simple design of HTML. Except for the hyperlink tag, these were strongly influenced by SGMLguid, an in-house Standard Generalized Markup Language (SGML)-based documentation format at CERN. Eleven of these elements still exist in HTML 4.



# 4. 正说HTML

&emsp;&emsp;经过自己动手，你现在已经有能力用文本编辑器写出一个简单页面了，这是个值得恭喜的进步！但是对于打算成为专业的Web前端开发者的人而言，这也仅仅是个开始。

## 4.1 标签

### 以你的第一个页面为例

&emsp;&emsp;在我们的例子里，\<b\>和\</b\>两个符号把一段纯文本（plain text）内容包围起来。

        <!--第一个符号是开始标签，它以"<"开头,以">"结尾，两个符号之间的"b"是标签名称-->
        <b>
        <!--这一段纯文本是文档显示出来的内容-->
        Hello, This is my first web page!
        <!--结束标签,它的写法和开始标签有一点不同-->
        </b>

&emsp;&emsp;为了显得更加专业，从今以后，我们要把\<b\>和\</b\>都称作HTML标签！标签的名字英文字母b在这里代表的含义是**bold**,假如你没有听说过这个单词，请自觉去查英汉字典。

&emsp;&emsp;现在，请动手把页面上的\<b\>和\</b\>两个标签删除掉，重新在浏览器里刷新页面，观察页面有没有变得不一样。
        
&emsp;&emsp;正如传统出版行业里面负责校对手稿的人（markup man）在文字旁边打标记（markup）是为了告诉排版工人这个文字最终需要按照什么样的字号、字体、格式显示出来，我们在HTML文档里这样写，也是为了告诉浏览器：我想要这段文字内容显示成我想要的样子。

&emsp;&emsp;你会发现，当把页面的开始标签、结束标签去掉后，网页上显示出来的句子，字体没原来那么粗了。没错！在这里我使用标签的目的，就是为了给文字加粗体显示。

### HTML文档基本元素

&emsp;&emsp;从最直观的感觉出发，我们可能会得出结论：HTML文档最基本的单位是HTML标签。实际上，标签所起到的作用，只是告诉浏览器，作者希望被标记的文字内容按照什么样子显示，换句话说就是它定义了内容的显示效果。

&emsp;&emsp;内容和被标记的内容组合在一起组成了HTML元素，我这里专门画图解释了这件事情。

## 4.2 文档结构

&emsp;&emsp;所谓HTML文档结构，就是页面上的HTML元素组合方式，它是HTML标准中最重要的组成部分，由Ｗ３Ｃ组织制定、发布。所有浏览器都遵守标准来解读和显示网页。

&emsp;&emsp;完整的HTML文档需要把\<html\>作为起始标签，它告诉浏览器：你要转备好开始展示网页了！当浏览器看到一个\</html\>结束标签的时候，它会知道，HTML文档内容到此结束，网页里面的内容显示完了！它们分别扮演了一个头和脚的角色。

 ![peter-html-structure](/public/img/html/peter-html-structure.png){:width="500px" height="600px"} 

&emsp;&emsp;在头和脚之间，整个HTML文档主要分为两部分：

1. head元素：这里用来包含网页的标题，还有一些其他重要的公共信息（元数据，metadata）
2. body元素： 这里包含了你打算在网页上显示的内容和信息

## 4.3 HTML4初降世

&emsp;&emsp;没有一种技术在它刚出现的时候就是完美无缺、尽如人意。在HTML语言随着互联网诞生的前几年，人们出于网页开发的需要就对它进行好几次
修订、规范，版本号从1.0到3.2一路飙升。直到１９９７年W3C组织发布HTML４.０版本，才算有了供全体HTML开发者参考遵守的统一的语言规范。

 ![html-brief-history](/public/img/html/html-brief-history.png){:width="600px" height="500px"}

### HTML4进化历程

&emsp;&emsp;在HTML4.0之前的年代，人们在制作网站的过程中还不知道要遵循什么统一标准。举一个简单的例子，比如说有两个大学教授，它们想要给学生们做一个分享课程信息的网站：

1. 首先，根据自己的需要定义几个只有他们两个才看得懂的的标签
2. 写好单个HTML网页,用超文本把网页连接成一个网站
3. 把网站放到万维网上
4. 自己写一个网站浏览器，送给小范围内，需要查看课程信息的学生或者同事

&emsp;&emsp;肇始之初的万维网世界，网页在小范围内创作，网站被人数不多的小群体浏览。以创作和浏览网站为目的，不同的人群，群体内部流行一种彼此能够理解的网页语言（HTML）,群体之间反而就像原始社会的部落之间那样，不需要沟通、交流，真的就是一副小国寡民、老死不相往来的古朴景象。

&emsp;&emsp;在HTML语言发展早期，专属于一个社区的HTML写法必然会被专为这个社区定制的网页浏览器准确展示。当万维网上每一个社区都用HTML方言创作网站的时候，适合一个网站的网页浏览器，在多数情况下不能正常显示另外一个网站的内容！更不必说，没有一个人可以做到，在不用更换浏览器的情况下，轻松查看全部网页！随着时间的推移，人们发现“言语不通”真的是很严重的问题！HTML语言规范呼之欲出！

&emsp;&emsp;1997年，W3C组织在总结HTML累积多年的发展成果后，发布了全面规范的HTML4.0语言标准，这是HTML语言标准历史上最大规模的一次进化。至此，人们告别了没有HTML社区标准的混乱时期。这件事在HTML标准史上，意义恰好可以与中国历史上秦朝人统一汉字写法相类比。道理很明显，只有在能够被所有人读懂的文字写法被记录下来，“藏之名山，传之其人”，千秋万世，代代流传以后，人们才可以在更长的时间、空间里面，学习和推广这门语言。HTML语言标准的统一，直接加速了网站技术的应用普及。

 ![chinese_char_uniform](/public/img/html/chinese_char_uniform.jpg)

>先秦古字经过这次整理之后，字样结构得以定型，这对贯彻法令、传播文化起了重大作用。我国地广人多，各地方言迥异，后来历史上又多次出现分裂割据的局面，而文字的统一，成为维系中华民族历史发展进步的一条生动鲜明的脉络。长期以来，我国封建社会高度发展的经济和文化屹立在世界文明的前列，这与秦朝大一统的开创之功有着不可分割的联系。

## 4.4 什么是XHTML

&emsp;&emsp;XHTML的全称是**eXtensible Hyper Text Markup Language**,中文全名：**可扩展超文本编辑语言**。它是W3C组织继HTML4标准之后对超文本标记语言进行的一次重大改进。

 ![html-xhtml-history](/public/img/html/html-xhtml-history.png)

### 更严格的语法规则

&emsp;&emsp;请你打开W3C验证服务，用HTML4和XHTML1.0两种Doctype类型进行检验下面一段示例代码：

    <html>
    <head>
    <title>This is bad HTML</title>
    <BODY>
    　　<h1>Bad HTML
    　　<p>This is a paragraph
　　　　　　<b><i>This text is bold and italic</b></i>
    </BODY>

&emsp;&emsp;从上面的除错过程中，你应该对XHTML语法的严格性有了第一手的直接体验。关于XHTML的这个特征，我们会在后续的章节进一步介绍引申。

&emsp;&emsp;W3C设计XHTML的目的就是要把严格的XML语法引入到网页开发过程中，顺便让HTML也可以获得XML那样强大的扩展性,在多个平台上得到良好兼容。在准备制定XHTML2.0标准的时候，他们非常激进地做出一个决定，“这个新的XHTML标准将不考虑兼容之前版本HTML语言写成的全部网页”！换句话说，如果所有的浏览器厂商完全采用它们的标准，所有的网站都将无法正常显示！当然，如果有人愿意重写全部网站，也还可以解决问题。

### XHTML2.0孤独老死

&emsp;&emsp;浏览器厂商最关心的是市场占有率。如果有一个厂商首先采用XHTML2.0标准,他的用户们就非常恼火地打电话来质问:“我安装过你们最新版本的浏览器之后，连一个网页都打不开、看不了，这是怎么回事？你现在要是不给我解决问题，我只好把你们的浏览器卸掉了！”

&emsp;&emsp;如果是你，你觉得跟用户长篇大论XHTML2.0标准会有意义吗？没有一个浏览器厂商愿意面对这样的尴尬和危险。这也是XHTML2.0不能被市场接纳，只能孤独死去原因。

&emsp;&emsp;下面是题目为[XHTML 2 Dies a Lonely Death, Makes Room For HTML 5](https://www.wired.com/2009/07/the_w3c_buries_xhtml_2dot0_html_5_is_the_future_of_the_web/)的文章对XHTML2之死富有历史意义的描述：

>The supposed successor to XHTML 1.x and the markup language that was once hailed as the next evolutionary step for the web has for all intents and purposes been dead for years. All the W3C has done is give it a proper headstone. And with the burial complete, the W3C can put all its efforts into the real future of the web — HTML 5...

## 4.5 HTML5

&emsp;&emsp;W3C组织深陷于他们自己精心炮制的美梦、无法自拔的时候，几个来自苹果公司、Mozilla基金会和Opera软件的个人不满意现状，在2004年发起了一个WHATWG的组织，意在推出他们心目中的下一代HTML标准，他们把自己的下一代HTML命名为HTML5。

&emsp;&emsp;考虑到自己的XHTML2.0标准陷入困境，在2007年，W3C新成立的HTML工作组接受WHATWG成员提出的建议，把WHATWG已经完成的HTML5标准作为下一代HTML技术的工作起点，双方联合起来共同创作、推出正式的下一代HTML标准。

 ![html5_intro_icon](/public/img/html/w3c_abandon_xhtml_desc.jpg){:width="600px" height="500px"}

### 狭义的HTML5

&emsp;&emsp;严格地讲，HTML5是HTML语言在HTML4.01和XHTML1.0基础上的最新一次重大版本升级，这次升级中HTML语言增加了许多反映前端发展趋势、满足人们现实应用要求的新特性。它完全兼容以前所有版本HTML语言，包括XHTML1.0。

&emsp;&emsp;目前制定HTML5标准的工作还没有结束。虽然厂商们都在极力炒作，但是实际上还没有一家浏览器完全支持这个标准。

 ![html5_is_coming](/public/img/html/html5_superman.png){:width="400px" height="300px"}

### 广义的HTML5

&emsp;&emsp;HTML5与HTML4最大的变化在于，诞生于移动互联网时代的HTML5标准必须要满足广大网民想要从他们各种屏幕尺寸不同、操作系统类型不同的移动设备上浏览网站的需要。随着全世界范围内互联网快速普及，以HTML为代表的Web前端技术更加无处不在，一部分有想法的人就考虑能不能基于Web前端技术创建一个运行在所有操作系统平台上的应用程序。虽然HTML5应用程序取代Android或者iOS本地应用还有很长一段路要走，但是我们已经可以在这个方向上看到许多令人激动的尝试，这是一个被业内人士普遍看好的发展方向。

&emsp;&emsp;好好想想，不久将来的那么一天，你正在带领一个创业团队：你再也不用像以前人们做的那样招聘两套人马，给一个应用程序同一时间发布Android和iOS两个平台、还要考虑几十种型号的屏幕尺寸，这将节省多少时间、金钱，又将降低多少商业风险！

 ![html5_cross_platform](/public/img/html/html5_cross_platform.png){:width="400px" height="300px"}

&emsp;&emsp;今日的Web前端开发者在运用HTML5开发网页应用的时候，还需要用到了许多辅助性的标准或者技术，这其中包括CSS3、新一代JavaScript。也其他名目繁多、令人目不暇接的技术标准，比如JSON，Ajax等等，我都不能一一列举。
